## Description de l'application

&#8203;<Highlight color="tip">3 points</Highlight> Décrivez votre application en quelques paragraphes.

- Quel est le but de l'application ?
- À qui s'adresse-t-elle ?
- Quelles sont les fonctionnalités principales ?

</Column>

<Column>

## Dessins en wireframe

&#8203;<Highlight color="tip">4 points</Highlight> Dessinez les écrans principaux de l'application (à la main ou avec un outil comme Figma, Excalidraw, etc.).

- Minimum **4 écrans** différents
- La navigation entre les écrans doit être visible (flèches ou légende)
- Chaque écran doit être légendé (nom des éléments visibles)

</Column>

</Row>

<Row>

<Column>

## User Stories

&#8203;<Highlight color="tip">3 points</Highlight> Rédigez au moins **10 user stories** sous la forme :

> En tant que [rôle], je veux [fonctionnalité] afin de [bénéfice].

Les user stories doivent couvrir les fonctionnalités décrites et les écrans dessinés.

</Column>

<Column>

## Qualité du document

&#8203;<Highlight color="danger">2 points négatifs</Highlight> Le document doit être clair, bien structuré et sans fautes.
Ce pointage fonctionne en négatif.

- Fautes d'orthographe ou de grammaire importantes → **-1 point**
- Document désorganisé ou difficile à lire → **-1 point**

</Column>
