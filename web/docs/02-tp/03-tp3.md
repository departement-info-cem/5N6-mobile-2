## Authentification Firebase

&#8203;<Highlight color="tip">6 points</Highlight> L'application utilise **Firebase Auth** avec courriel et mot de passe.

- Écran d'inscription (courriel + mot de passe)
- Écran de connexion
- Déconnexion accessible depuis l'application
- Gestion des erreurs (courriel déjà utilisé, mot de passe invalide, etc.)
- L'utilisateur reste connecté entre les sessions (persistance)

</Column>

<Column>

## Firebase Firestore

&#8203;<Highlight color="tip">8 points</Highlight> Les données principales de l'application sont stockées dans Firestore.

- Lire et afficher les données de l'utilisateur connecté
- Créer un nouveau document
- Modifier ou supprimer un document (selon le contexte de votre application)
- Les données sont associées à l'utilisateur connecté (par UID)

</Column>

## Élément au choix

&#8203;<Highlight color="tip">4 points</Highlight> Implémentez **un** des éléments suivants :

## Fonctionnement global

&#8203;<Highlight color="danger">5 points négatifs</Highlight> Votre application doit fonctionner sans plantage et correctement.
Ce pointage fonctionne en négatif.

- Plantage de l'application → **-1 point**
- Interface illisible ou débordements → **-1 point**
- Données Firebase incohérentes → **-1 point**
- Authentification non sécurisée (données non liées à l'UID) → **-1 point**
- Autre cas problématique → **-1 point**
