## User Stories réalisées

&#8203;<Highlight color="tip">10 points</Highlight> Les user stories de votre TP1 qui n'étaient pas dans le MVP sont implémentées.

- Chaque user story réalisée est clairement identifiable dans l'application
- La fonctionnalité fonctionne de façon cohérente avec le reste de l'application
- Les points attribués dépendent du nombre et de la complexité des stories réalisées

</Column>

<Column>

## Présentation orale

&#8203;<Highlight color="tip">5 points</Highlight> Vous présentez votre application lors de la séance 14.2 (~5 minutes).

- Démonstration des fonctionnalités principales
- Explication des défis rencontrés et des solutions apportées
- Questions du professeur

## Qualité du code

&#8203;<Highlight color="tip">3 points</Highlight> Le code est bien organisé et lisible.

- Les widgets sont extraits dans des fichiers séparés (pas tout dans `main.dart`)
- Pas de code mort ou commenté inutilement
- Nommage cohérent des variables et des fonctions

</Column>

<Column>

## Fonctionnement global

&#8203;<Highlight color="danger">5 points négatifs</Highlight> Votre application doit fonctionner sans plantage et correctement.
Ce pointage fonctionne en négatif.

- Plantage de l'application → **-1 point**
- Interface illisible ou débordements → **-1 point**
- Données Firebase incohérentes → **-1 point**
- Fonctionnalités incomplètes ou brisées → **-1 point**
- Autre cas problématique → **-1 point**
